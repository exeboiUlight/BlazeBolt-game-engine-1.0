# BlazeBolt Game Engine 1.0

Легковесный 2D игровой движок на C++ с Lua скриптами и встроенным редактором проектов.

## Особенности

- **Lua API** — простое создание игр на Lua 5.4
- **Быстрая графика** — OpenGL 3.3 + stb_image для текстур
- **Физика** — встроенный 2D физический движок
- **Встроенный редактор** — Hub на Python/PyQt6 для управления проектами
- **Кэширование текстур** — автоматическое переиспользование текстур по пути
- **Система сцен** — загрузка/выгрузка сцен с коллбэками `On<Scene>Load/Unload`

## Быстрый старт

### Требования

- **Windows** (x64)
- **Python 3.8+** (для сборки и Hub)
- **PyQt6** (для Hub): `pip install PyQt6`
- **CMake 3.10+**
- **MinGW-w64** или **Visual Studio** с C++ компилятором

### Сборка движка

```bash
# Клонировать репозиторий
cd "BlazeBolt game engine 1.0"

# Сборка через build.ul (создает bin/versions/game.exe)
python src/build.ul

# Или через CMake
mkdir build && cd build
cmake ..
make
```

### Запуск Hub (менеджера проектов)

```bash
cd hub
python main.py
```

## Создание игры

### 1. Создание проекта

1. Откройте **Hub** (`python hub/main.py`)
2. Нажмите **"Создать проект"**
3. Выберите папку проекта
4. Hub автоматически создаст структуру:
   ```
   MyGame/
   ├── scenes/           # JSON файлы сцен
   │   └── main.json    # Главная сцена
   ├── scripts/         # Lua скрипты
   ├── textures/        # PNG текстуры
   └── game.exe         # Копия движка
   ```

### 2. Редактирование сцены

В Hub нажмите **"Открыть"** на проекте:

- **Левая панель**: Файлы проекта и объекты сцены
  - Создавайте папки и файлы кнопками 📁+ / 📄+
  - Удаляйте выбранное кнопкой 🗑
  - Двойной клик по `.json` файлу загружает сцену
  - Двойной клик по `.lua` открывает скрипт в редакторе

- **Центральная панель**: Inspector и Code Editor
  - **Inspector**: Настройка свойств выбранного объекта
    - Имя, Тип, Позиция (X, Y), Размер, Текстура, Скрипт
    - Нажмите **"✅ Применить"** для сохранения изменений
  - **Code Editor**: Редактор кода с подсветкой синтаксиса Lua
    - Поддержка тёмной и светлой темы
    - Кнопка **"💾 Сохранить"** сохраняет изменения в файл

- **Правая панель**: Управление сценой
  - Имя файла сцены (по умолчанию "main")
  - **"💾 Сохранить сцену"** — сохраняет в `scenes/<имя>.json`
  - **"📂 Загрузить сцену"** — загружает указанную сцену

### 3. Добавление объектов

В редакторе сцены:
1. Нажмите **"➕"** (добавить объект)
2. Выберите объект в списке слева
3. Настройте свойства в Inspector:
   - **Типы объектов**: Sprite2D, Label, Camera2D, Area2D, Node2D, и др.
   - **Позиция**: X, Y (в экранных координатах)
   - **Размер**: Width, Height
   - **Текстура**: путь к PNG файлу
   - **Скрипт**: путь к Lua файлу

### 4. Написание скриптов

Пример минимального скрипта (`scripts/player.lua`):

```lua
function _ready()
    print("Игрок готов!")
end

function _process(dt)
    -- Вызывается каждый кадр
    local speed = 0.5
    if BlazeBolt.IsKeyPressed(Keys.W) then
        -- Движение вверх
    end
end
```

### 5. Запуск игры

В Hub нажмите **"▶️ Запустить"** на проекте, или запустите `game.exe` в папке проекта.

## Lua API

Полная документация доступна в [DOCUMENTATION.md](DOCUMENTATION.md).

### Основные функции

```lua
-- Создание спрайта
local sprite = BlazeBolt.CreateSprite("textures/player.png", 0, 0)

-- Установка размера
BlazeBolt.SpriteSetSize(sprite, 0.3, 0.3)

-- Центрирование точки привязки
BlazeBolt.SpriteSetOrigin(sprite, 0.5, 0.5)

-- Проверка нажатия клавиш
if BlazeBolt.IsKeyJustPressed(Keys.SPACE) then
    print("Прыжок!")
end

-- Создание текста
local fps = BlazeBolt.CreateText("fonts/arial.ttf", "FPS: 60", -0.8, 0.8, 24)
```

### Система координат

Движок использует OpenGL NDC:
- **Центр экрана**: `(0, 0)`
- **Верхний правый угол**: `(1, 1)`
- **Нижний левый угол**: `(-1, -1)`

Размеры задаются в NDC: `0.1` = 10% от высоты экрана.

### Жизненный цикл

```lua
function Start()       -- Вызывается один раз при запуске
function Update(dt)    -- Вызывается каждый кадр (dt в секундах)
function Draw()        -- Вызывается для отрисовки каждый кадр
function End()         -- Вызывается при завершении работы

-- Callbacks сцен (автоматически вызываются при смене сцены "Menu")
function OnMenuLoad()    -- При загрузке сцены "Menu"
function OnMenuUnload()  -- При выгрузке сцены "Menu"
```

Hub поддерживает две темы:
- **Тёмная** (Dracula) — по умолчанию
- **Светлая** (Clean Modern) — переключается кнопкой ☀️/🌙

## Система сцен

Сцены сохраняются в JSON формате:

```json
{
    "name": "main",
    "objects": [
        {
            "name": "Player",
            "type": "Sprite2D",
            "x": 0.0,
            "y": 0.0,
            "width": 0.2,
            "height": 0.2,
            "texture": "textures/player.png",
            "script": "scripts/player.lua"
        }
    ]
}
```

При загрузке сцены движок:
1. Выгружает текущую сцену (вызывает `On<OldScene>Unload`)
2. Загружает новую сцену (вызывает `On<NewScene>Load`)
3. Создает объекты и привязывает скрипты

## Полезные советы

1. **Текстурный кэш**: Текстуры кэшируются по пути файла. Повторное использование `BlazeBolt.SpriteSetTexture()` с тем же путем не загружает текстуру заново.

2. **Горячие клавиши в редакторе**:
   - `F11` — полный экран (в игре)

3. **Структура проекта**: Рекомендуется хранить текстуры в `textures/`, скрипты в `scripts/`, сцены в `scenes/`.

4. **Отладка**: Используйте `print()` в Lua скриптах — вывод появится в консоли.
