# BlazeBolt Game Engine 1.1

Лёгкий и оптимизированный игровой движок на C++ с поддержкой Lua-скриптинга.

---
## Требования к build машине

1. g++ >=15.3
2. cmake >=3.10
   
*в этом репозитории используются submodule
`git submodule update --init --recursive`
### Операционные системы на которых тестировалось
Полностью работает:
  arch linux
  windows 10-11
Не удалось собрать:
  debian 13 (только в тестовой версии g++-15)

### Зависимости

| Библиотека | Назначение |
|---|---|
| GLFW | Окна и ввод |
| GLAD | OpenGL-загрузчик |
| FreeType | Рендеринг шрифтов |
| Lua 5.4 | Скриптовый движок |
| OpenAL | Звук |
| ws2_32 (Windows) | Сетевые сокеты (TCP/UDP) |

---

## Идеология

1. **Минимальный вес** и **максимальная оптимизация** — движок работает практически на любом оборудовании.
2. большая работа с модами.
3. удобная работа с движком позволяет воплощать **любые идеи**, а портабельность даёт возможность делать сразу и для линукс и для виндовс.

---

## Возможности

- 2D-спрайты с поддержкой текстурных атласов
- Анимированные спрайты (GIF)
- Спрайтовые батчи для оптимизированной отрисовки
- Система частиц (GPU-инстансырование)
- Тайловые карты (Tileset2D)
- 2D-физика (круги, прямоугольники)
- Камера с масштабированием и вращением
- Система освещения (точечные + фоновые источники)
- Пользовательские шейдеры
- Сетевое взаимодействие (TCP/UDP)
- Процедурные шумы (Перлин, симплекс, FBM, Domain Warp)
- Математические типы (Vector2/3/4, Matrix3x3)
- Управление порядком отрисовки слоёв
- Многоскриптовая архитектура с переключением сцен

## Решение проблем

### Ошибка `undefined reference to __imp_WSAStartup` (Windows)

При компиляции с сетевым модулем линковщик не находит Winsock-функции.
Убедитесь, что `-lws2_32` добавлен в флаги линковщика.
